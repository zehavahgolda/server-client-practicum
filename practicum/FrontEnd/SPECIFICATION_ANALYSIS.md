## 📦 מודולים עיקריים

### 1. **Dashboard Module** - דשבורד ניהולי
**תפקיד:** הצגת תמונת מצב כללית עם KPIs ואזהרות
- חישוב 4 KPIs עיקריים
- אזהרות על בעיות קריטיות
- דוחות ויזואליים (עקומות בחירה, דוגנוטים)
- drill-down ממושכלל לכרטיסיות

### 2. **Employees Module** - ניהול עובדים
**תפקיד:** ניהול פרופילי עובדים ותכנון קיבולתם
- יצירה, קריאה, עדכון, מחיקת עובדים
- הצגת פרופיל עובד מלא
- חישוב יתרת קיבולת
- סטטוס סכנה (זמינות תקינה / קיבולת נמוכה / עומס יתר)

### 3. **Systems Module** - ניהול מערכות
**תפקיד:** ניהול מערכות ודרישות הקיבולת שלהן
- ניהול מערכות בתחום הטכנולוגיה
- הצגת סטטוס קיבולה (At Risk / Shortage / OK / Excess)
- קיבוץ לפי מצב או פער קיבולת
- דוחות על עובדים משויכים

### 4. **Allocations Module** - ניהול הקצאות
**תפקיד:** שיבוץ עובדים למערכות
- תיבת דו-שיח עם חיפוש עובדים
- בחירה מרובה של עובדים
- ולידציה של קיבולת
- עדכון מידי של מדדים

### 5. **Budget Module** - ניהול תקציבים
**תפקיד:** בקרה על תקציבים לפי תחומים
- חישוב עלויות לפי קטגוריית עובד
- ניתוח חריגות תקציביות
- דוחות עובדים מובילים בעלויות
- חישוב FTE (יחידות עבודה שלמות)

### 6. **Categories Module** - קטגוריות מקצועיות
**תפקיד:** ניהול קטגוריות התמחות
- הגדרת קטגוריות (פיתוח, UX/UI, יישום, PM)
- קיבולת נדרשת וקיימת
- תוך-תחומים וצבעים אישיים

### 7. **Changes Module** - ניהול שינויים
**תפקיד:** תיעוד השפעות שינויים תפעוליים
- Timeline שינויים
- סיווג לפי חומרה (info / warn / danger / ok)
- תיעוד השפעות על קיבולת

## 🔍 ישויות (Entities)

### **1. Employee - עובד**

```javascript
{
  id: number,                          // מזהה ייחודי
  name: string,                        // שם עובד
  category: string,                    // קטגוריה מקצועית
  subcategory: string,                 // תת-תחום (Backend, Frontend, DBA, DevOps...)
  manager: string,                     // שם המנהל הישיר
  yearlyCapacity: number,              // קיבולת שנתית בחודשי עבודה (בד"כ 10-12)
  futureEvents: string,                // אירועים עתידיים (חופשה, קורס, מילואים)
  allocations: {
    [systemId]: number                 // חודשי עבודה מוקצים למערכת
  }
}
```

**דוגמה:**
```javascript
{
  id: 1,
  name: "נועה ארז",
  category: "פיתוח",
  subcategory: "Backend",
  manager: "עידו טל",
  yearlyCapacity: 12,
  futureEvents: "מילואים ברבעון ג׳",
  allocations: { servicenow: 4, gis: 0, internal: 3, bi: 2, cyber: 1 }
}
```

### **2. System - מערכת**

```javascript
{
  id: string,                          // מזהה ייחודי
  name: string,                        // שם המערכת
  required: number,                    // חודשי עבודה נדרשים בשנה
  totalBudget: number,                 // סה"כ תקציב (בש"ח)
  statusHint: "ok" | "shortage" | "excess" | "risk"  // סטטוס תכנון
}
```

**דוגמה:**
```javascript
{
  id: "servicenow",
  name: "ServiceNow",
  required: 42,
  totalBudget: 4843,
  statusHint: "ok"
}
```

### **3. Category - קטגוריה מקצועית**

```javascript
{
  title: string,                       // שם קטגוריה
  colorVar: string,                    // CSS variable לצבע
  children: string[],                  // תחומי משנה
  required: number,                    // קיבולת נדרשת כללית
  existing: number                     // קיבולת קיימת כללית
}
```

**דוגמה:**
```javascript
{
  title: "Development",
  colorVar: "--cat-dev",
  children: ["Backend", "Frontend", "DBA", "DevOps"],
  required: 95,
  existing: 82
}
```

### **4. Change - שינוי תפעולי**

```javascript
{
  date: string,                        // תאריך (DD/MM/YYYY)
  type: "info" | "warn" | "danger" | "ok",  // סיווג
  title: string,                       // כותרת השינוי
  impact: string                       // תיאור ההשפעה על קיבולת
}
```

**דוגמה:**
```javascript
{
  date: "03/01/2026",
  type: "info",
  title: "מערכת BI חדשה נוספה",
  impact: "+8 חודשי עבודה נדרשים, בעיקר אנליסטים ו-DBA"
}
```

## 🔄 זרימות עיקריות (Business Flows)

### **זרימה 1: צפייה בדשבורד ניהולי**

```
[1] משתמש פתח את האפליקציה
    ↓
[2] טעינת נתונים מ-SessionStorage
    (עובדים, מערכות, קטגוריות, שינויים)
    ↓
[3] חישוב KPIs:
    - מערכות בסיכון (gap > 4)
    - פער קיבולת כולל (סכום חסכים)
    - שיעור ניצול (מנוצל / קיבולת)
    - ציון בריאות כללי (0-100%)
    ↓
[4] חישוב אזהרות:
    - בעיות קריטיות למערכות
    - עובדים בקיבולת נמוכה
    - קטגוריות בחסכות
    ↓
[5] הצגת דוחות ויזואליים:
    - עקומות ביקוש לפי מערכת
    - דוגנוט התפלגות עובדים
    ↓
[6] אופציה drill-down:
    - קליק על KPI → יישום פילטר
    - מעבר לרשימה מסוננת
    - עדכון מידי של תוצאות
```

### **זרימה 2: הקצאת עובד למערכת**

```
[1] משתמש נכנס לפרופיל מערכת
    ↓
[2] לחץ כפתור "שיבוץ עובדים"
    ↓
[3] פתיחת תיבת דו-שיח עם:
    - שדה חיפוש עובד
    - סינון לפי תחום, מנהל, שם
    - הצגה של:
      * שם עובד
      * קטגוריה ותת-תחום
      * יתרת קיבולת
      * מספר מערכות פעילות
    ↓
[4] בחירה של עובדים (multi-select)
    ↓
[5] כפתור "שיבוץ נבחרים"
    ↓
[6] וולידציה:
    - בדיקה אם לעובד יש קיבולת כافה
    - בדיקה היכן כבר משויך
    ↓
[7] שמירה בנתונים:
    - עדכון allocations של עובד
    - עדכון allocations של מערכת
    ↓
[8] חישוב מחדש:
    - יתרת קיבולת עובד
    - פער מערכת
    - תקציב מערכת
    - סטטוס מערכת
    ↓
[9] רנדור מחדש של ממשק
    ↓
[10] הצגת ודאות הצלחה
```

### **זרימה 3: ניתוח תקציב של מערכת**

```
[1] משתמש כנס לפרופיל מערכת
    ↓
[2] בחר בלשונית "תקציב"
    ↓
[3] מערכת מחשבת עבור כל עובד:
    חודשים מוקצים × תעריף = עלות
    
    תעריפים:
    - פיתוח: 260 ש"ח
    - UX/UI: 230 ש"ח
    - יישום: 210 ש"ח
    - ניהול פרויקטים: 220 ש"ח
    ↓
[4] קיבוץ לפי תת-תחום:
    Backend: 3 עובדים, 12 חודשים, 3,120 ש"ח
    Frontend: 2 עובדים, 8 חודשים, 1,840 ש"ח
    DBA: 1 עובד, 4 חודשים, 1,040 ש"ח
    ↓
[5] חישובי סיכום:
    - סה"כ חודשים מוקצים
    - FTE (יחידות עבודה שלמות)
    - ממוצע ניצול חודשי
    - כיסוי מול דרישה
    ↓
[6] בדיקה של סטטוס תקציבי:
    עלות כוללת vs סה"כ תקציב
    ↓
[7] הצגה של:
    - טבלת פירוט לפי תחום
    - עובדים המובילים בעלויות
    - הערות בקרה (חריגה/עודף/מאוזן)
```

### **זרימה 4: סינון וקיבוץ עובדים**

```
[1] משתמש בחר קיבוץ:
    - "כל העובדים" - ללא קיבוץ
    - "קטגוריה" - קבוצות לפי קטגוריה מקצועית
    - "מנהל" - קבוצות לפי מנהל ישיר
    ↓
[2] מחשוב קבוצות עם סטטיסטיקות:
    - מספר עובדים בקבוצה
    - קיבולת כוללת / מנוצלת
    - מספר עובדים בלחץ
    ↓
[3] הצגה עם collapse/expand:
    - כותרת קבוצה עם סטטיסטיקות
    - סרגל צבעון בהתאם לסטטוס
    - רשימת עובדים בתוך הקבוצה
    ↓
[4] ניתן להפעיל סינונים נוספים:
    - לפי שנה
    - לפי קטגוריה (רק כאשר לא מקובצים)
    - לפי מנהל (רק כאשר לא מקובצים)
    - לפי מערכת
    - חיפוש טקסט חופשי
    ↓
[5] עדכון מידי של התוצאות
    ↓
[6] ניתן לפתוח פרופיל עובד מיוחד
```

## 🎯 Key Performance Indicators (KPIs)

### **KPI 1: מערכות בסיכון (Systems at Risk)**
```
הגדרה: מספר מערכות עם gap > 4 חודשי עבודה
ערך: count
סטטוס: 
  - אדום (critical) אם > 0
  - ירוק (healthy) אם = 0
משמעות: מערכות בהן קיים מחסור קריטי
פעולה: דורש החלטה מיידית (תגבור או שינוי תכולה)
```

### **KPI 2: פער קיבולת כולל (Total Capacity Gap)**
```
הגדרה: סכום כל הפערים החיוביים של מערכות
ערך: חודשי עבודה
סטטוס:
  - אדום (critical) אם > 0
  - ירוק (healthy) אם = 0
משמעות: כמה חודשי עבודה חסרים באופן כולל
פעולה: צריך להיות זהה ל"בעיות קריטיות" בדשבורד
```

### **KPI 3: ניצול קיבולת (Utilization Rate)**
```
הגדרה: (סכום הקצאות / סה"כ קיבולת) × 100%
ערך: אחוז (0-100%)
צבע:
  - ירוק: 0-75%
  - כתום: 75-90%
  - אדום: >90%
משמעות: כמה אחוז מהקיבולת הכללית מנוצל
פעולה: 
  - <75% = יש עודף קיבולת
  - 75-90% = ניצול בריא
  - >90% = בקצה הקיבולת
```

### **KPI 4: ציון בריאות כללי (Health Score)**
```
הגדרה: ממוצע של:
  1. (מערכות מאוזנות / כל המערכות)
  2. (1 - shortage_ratio)
  3. (1 - low_capacity_employees_ratio)

ערך: אחוז (0-100%)
צבע:
  - ירוק: ≥75%
  - כתום: 50-74%
  - אדום: <50%
משמעות: מצב בריאות כללי של המערכת
```

### **קשר בין KPIs:**

```
Systems at Risk = count(gap > 4)
        ↓
Total Gap = sum(positive gaps)
        ↓
Utilization Rate = allocated / capacity
        ↓
Health Score = average(system_balance, gap_ratio, employee_balance)
```

## 💰 מודל תקציבי

### **עלות בסיסית**

```
עלות חודש עבודה לעובד = מספר חודשים × תעריף עפ"י קטגוריה

תעריפים קבועים בש"ח (שקל חדש):
┌───────────────────────┬──────────┐
│ קטגוריה              │ תעריף    │
├───────────────────────┼──────────┤
```

## 📊 חישובים ליבה

### **1. יתרת קיבולת עובד**
```javascript
employeeGap = employee.yearlyCapacity - SUM(allocations)

דוגמה:
נועה ארז: 12 - (4 + 0 + 3 + 2 + 1) = 12 - 10 = 2 חודשים יתרה
```

### **2. פער מערכת**
```javascript
systemGap = system.required - SUM(allocated_months)

דוגמה:
ServiceNow: 42 - 35 = 7 חודשים חסרים (At Risk)
```

### **3. תקציב מערכת**
```javascript
budgetUsed = SUM(months × rate_by_category)
budgetRemaining = totalBudget - budgetUsed
budgetDifference = budgetUsed - totalBudget

דוגמה:
Backend: 9 חודשים × 260 = 2,340 ₪
Frontend: 2 חודשים × 260 = 520 ₪
סה"כ: 2,860 ₪
```

### **4. כיסוי הקצאה**
```javascript
coveragePct = (allocatedMonths / requiredMonths) × 100%

דוגמה:
ServiceNow: (35 / 42) × 100 = 83.3% כיסוי
```

### **5. שיעור ניצול**
```javascript
utilizationRate = (SUM(all allocations) / SUM(all capacities)) × 100%

דוגמה:
כולל: 50 חודשים / 64 = 78% ניצול
```

### **6. ציון בריאות**
```javascript
healthFactors = [
  systemsBalanced / totalSystems,
  1 - (shortage / totalCapacity),
  1 - (lowCapacityEmployees / totalEmployees)
]
healthScore = AVERAGE(healthFactors) × 100%

דוגמה:
3 מתוך 5 מאוזנות = 0.6
1 - (12 / 64) = 0.81
1 - (2 / 8) = 0.75
Average = 0.72 → 72% ציון בריאות
```
